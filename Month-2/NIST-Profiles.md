NIST Profiles
Overview
NIST Profiles help organizations understand their current cybersecurity posture and define their desired future state.
Profiles are used to identify security gaps and create improvement plans.
A Profile is essentially a snapshot of an organization's cybersecurity capabilities.
What Is a Current Profile?
The Current Profile describes the organization's existing cybersecurity state.
It shows which controls, processes, and capabilities are currently implemented.
Example:
MFA = No
Logging = No
Backups = Yes
Incident Response Plan = No
This profile helps organizations understand where they are today.
What Is a Target Profile?
The Target Profile describes the desired cybersecurity state.
It represents the security goals the organization wants to achieve.
Example:
MFA = Yes
Logging = Yes
Backups = Yes
Incident Response Plan = Yes
This profile helps organizations understand where they want to be.
Gap Analysis
Gap Analysis compares the Current Profile with the Target Profile.
The purpose is to identify missing controls and areas for improvement.
Example:
Current State:
MFA = No
Target State:
MFA = Yes
Gap:
Implement MFA
Current State:
Logging = No
Target State:
Logging = Yes
Gap:
Implement logging and monitoring
Why NIST Profiles Matter
Profiles help organizations:
Understand their current security posture
Identify weaknesses
Prioritize improvements
Support risk management
Improve governance
Prepare for audits
Example
Current Profile:
MFA = No
Logging = No
Backups = Yes
Target Profile:
MFA = Yes
Logging = Yes
Backups = Yes
Incident Response Plan = Yes
Identified Gaps:
Enable MFA
Implement logging
Create an Incident Response Plan
Main Lesson
NIST Profiles help answer three critical questions:
Where are we now?
Where do we want to be?
What improvements are required to get there?
