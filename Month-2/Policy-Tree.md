# Policy Tree

## Overview

Organizations use a Policy Tree to structure security documentation.

A Policy Tree creates a hierarchy that connects high-level policies with standards, procedures, and guidelines.

This helps organizations maintain consistency and accountability.

---

# Policy Tree Structure

Policy
↓
Standard
↓
Procedure
↓
Guideline

---

# Policy

Policies define management expectations and objectives.

Example:

Information Security Policy

Purpose:

Protect company information and systems.

---

# Standard

Standards define mandatory requirements.

Example:

Password Standard

Requirements:

* Minimum 12 characters
* MFA required
* Password reuse prohibited

---

# Procedure

Procedures provide step-by-step instructions.

Example:

How to Change a Password

1. Open account settings.
2. Select Change Password.
3. Create a password that meets requirements.
4. Save changes.

---

# Guideline

Guidelines provide recommendations and best practices.

Example:

Use a password manager to create strong passwords.

---

# Example Policy Tree

Information Security Policy

├── Access Control Policy

│ ├── Password Standard

│ ├── MFA Standard

│ └── Enable MFA Procedure

│

├── Incident Response Policy

│

├── Data Protection Policy

│

└── Acceptable Use Policy

---

# Why Policy Trees Matter

Policy Trees help organizations:

* Organize documentation
* Support compliance
* Improve consistency
* Clarify requirements
* Simplify audits

---

# Main Lesson

Policy defines what must be done.

Standard defines requirements.

Procedure explains how to perform the task.

Guideline provides recommendations.

Together they create an effective governance framework.
