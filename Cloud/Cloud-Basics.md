# Cloud Basics

## Overview

Cloud computing allows organizations to use infrastructure, storage, networking, and software through the internet instead of managing everything locally.

Cloud services help companies:
- scale faster;
- reduce hardware costs;
- improve availability;
- simplify operations.

---

# What is Cloud?

Cloud is a system where companies rent computing resources through the internet.

Instead of buying physical servers and storing them locally, organizations use remote infrastructure provided by cloud providers.

Cloud resources may include:
- virtual servers;
- storage;
- databases;
- networking;
- applications;
- security tools.

# Google Cloud

Google Cloud Platform (GCP) is Google's cloud ecosystem.

It is known for:

-AI services;
-analytics;
-big data;
-Kubernetes support.
-API Basics

API stands for:

-Application Programming Interface

APIs allow applications and systems to communicate with each other.

Examples:

-weather apps;
-login systems;
-Telegram bots;
-online services.
-Exposed APIs

Exposed APIs are poorly protected or publicly accessible APIs.

Risks include:

-data leakage;
-unauthorized access;
-account abuse.

Security controls:

-authentication;
-authorization;
-monitoring;
-rate limiting.
-IAM

IAM stands for:

-Identity and Access Management

IAM controls:

-who can access systems;
-what permissions users have;
-what actions users can perform.

Examples:

-read-only access;
-administrator access;
-restricted permissions.
-Misconfiguration

Misconfiguration means incorrect or insecure cloud setup.

Examples:

-public databases;
-exposed storage buckets;
-weak permissions;
-disabled MFA.

Misconfiguration is one of the most common cloud security risks.

Cloud Service Models
IaaS — Infrastructure as a Service

The provider gives:

-infrastructure;
-networking;
-virtual servers.

The customer manages:

-operating systems;
-applications;
-configurations.

Examples:

-AWS EC2
-Azure Virtual Machines
-PaaS — Platform as a Service

The provider manages:

-infrastructure;
-operating systems;
-runtime environments.

The customer mainly deploys applications.

Examples:

-Heroku
-Google App Engine
-SaaS — Software as a Service

The provider manages almost everything.

Users simply use the software through the internet.

Examples:

-Gmail
-Google Docs
-Microsoft 365
-Service Type Examples
-Service	Type
-Gmail	SaaS
-Google Docs	SaaS
-AWS EC2	IaaS
-Heroku	PaaS
-Security Perspective

--Cloud security depends on:

-proper configuration;
-access control;
-MFA;
-monitoring;
-IAM policies.

Cloud is not automatically secure.

Organizations must configure and manage cloud environments properly.

# Main Lesson

* Cloud computing is not “magic internet.”

* Cloud is infrastructure, systems, storage, networking, and security delivered through the internet.
