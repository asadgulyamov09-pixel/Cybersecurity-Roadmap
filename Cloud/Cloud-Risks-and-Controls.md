# Cloud Risks & Controls

## Overview

Cloud environments provide scalability and flexibility, but they also introduce security risks that organizations must manage carefully.

Cloud security depends on:
- proper configuration;
- identity management;
- monitoring;
- governance;
- access control.

---

# Common Cloud Risks

## Misconfiguration

Incorrect cloud settings may expose:
- storage;
- systems;
- services;
- sensitive data.

Examples:
- public storage buckets;
- open ports;
- weak IAM policies.

---

## Weak IAM

Overprivileged accounts may lead to:
- unauthorized access;
- insider threats;
- privilege abuse.

---

## No MFA

Accounts protected only by passwords are vulnerable to:
- credential theft;
- phishing;
- account compromise.

---

## Exposed APIs

Poorly secured APIs may expose:
- sensitive data;
- internal systems;
- cloud services.

---

## No Logging & Monitoring

Without logs and monitoring:
- attacks may remain invisible;
- investigations become difficult;
- audit evidence may be missing.

---

# Security Controls

Important cloud security controls include:
- MFA;
- least privilege;
- monitoring;
- logging;
- encryption;
- patch management;
- access reviews.

---

# Risk Mapping

| Risk | Business Impact | Control |
|---|---|---|
| weak password | account compromise | MFA |
| public storage | data leakage | proper permissions |
| weak IAM | privilege abuse | least privilege |
| no logging | invisible attacks | monitoring |
| outdated systems | exploited vulnerabilities | patch management |

---

# Main Lesson

Cloud security is based on reducing risks through proper controls, governance, monitoring, and secure configurations.
