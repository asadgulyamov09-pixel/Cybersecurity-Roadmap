# Month 1 Learning Summary

## Overview

This document summarizes what I learned and practiced during my first month of studying cybersecurity.

My focus during this month was building strong foundations in networking, Linux, cyber threats, cloud security, governance, and risk management.

---

# Week 1 — Networking & Web Security

## Topics Studied

* OSI Model
* TCP/IP
* TCP vs UDP
* DNS
* DNS Spoofing
* HTTP vs HTTPS
* TLS
* MITM (Man-in-the-Middle)
* TCP Three-Way Handshake

## Key Lessons

I learned how devices communicate over networks and how security risks can appear at different layers of communication.

I also learned why encryption and secure communication protocols are critical for protecting data.

---

# Week 2 — Linux & Logging

## Topics Studied

* Linux basic commands
* chmod
* chown
* Linux permissions
* Linux file system
* SSH
* journalctl
* Logging and Monitoring
* Access Control

## Key Lessons

I learned how Linux permissions control access to files and directories.

I also learned how logs provide audit evidence and help security teams investigate incidents and suspicious activity.

---

# Week 3 — Threats & Risk Analysis

## Topics Studied

* Phishing
* Spear Phishing
* Ransomware
* DoS
* DDoS
* CIA Triad
* MITRE ATT&CK Basics
* Business Impact Analysis

## Key Lessons

I learned that cybersecurity is not only about technical attacks.

Every attack has business consequences such as:

* financial loss;
* reputational damage;
* operational disruption;
* data leakage.

I also learned how to connect threats, risks, impacts, and security controls.

---

# Week 4 — Cloud Security & Governance

## Topics Studied

* Cloud Computing Fundamentals
* AWS Basics
* Azure Basics
* IAM (Identity and Access Management)
* Shared Responsibility Model
* Cloud Risks
* Cloud Controls
* Least Privilege
* Access Reviews
* Cloud Governance

## Key Lessons

I learned that cloud security is a shared responsibility between the cloud provider and the customer.

I also learned how permissions, identity management, governance, and configurations directly affect cloud security.
